# Архитектура системы: Cloud Native URL Shortener

В данном документе описывается целевая архитектура микросервисов, схема базы данных и основные технические решения.

## 1. Логическая архитектура компонентов

Проект разделен на слабосвязанные компоненты (микросервисы), каждый из которых решает свою узкую задачу:

* **API Gateway (Laravel):** Точка входа для всех HTTP-запросов. Отвечает за генерацию коротких ссылок, валидацию данных и аутентификацию пользователей. Выполняет быстрый редирект (Hot Path), используя Redis для кэширования. События о переходах (кликах) асинхронно отправляются в очередь сообщений.
* **Message Broker (RabbitMQ):** Принимает события о кликах от API Gateway. Обеспечивает отказоустойчивость: если база данных или воркер временно недоступны, данные не потеряются.
* **Analytics Worker (Python/FastAPI):** Асинхронный консьюмер. Читает события из очереди RabbitMQ и батчами (пачками) записывает статистику в базу данных, снимая нагрузку на запись с основного шлюза.

## 2. Схема базы данных (PostgreSQL)

Ниже представлена ER-диаграмма основных сущностей проекта. 
*Примечание: между таблицами намеренно отсутствует жесткое ограничение внешнего ключа (Foreign Key constraint), чтобы обеспечить высокую скорость вставки в Append-Only таблицу статистики и избежать блокировок.*

```mermaid
erDiagram
    links ||--o{ click_stats : "логическая связь по short_code"
    links {
        bigint id PK
        varchar(15) short_code UK
        text original_url
        bigint user_id FK
        timestamp created_at
        timestamp updated_at
    }
    click_stats {
        bigint id PK
        varchar(15) short_code IDX
        timestamp clicked_at
        varchar(45) ip
        text user_agent
        text referer
        varchar(2) country
    }
```

### Детализация таблиц

#### Таблица `links` (Основная сущность)
Хранит информацию о сокращенных ссылках. Оптимизирована для быстрого чтения при редиректах.

| Поле | Тип данных | Индекс | Описание |
| :--- | :--- | :--- | :--- |
| `id` | bigint | Primary Key | Уникальный идентификатор |
| `short_code` | varchar(15) | **Unique** | Короткий код (используется для поиска) |
| `original_url` | text | - | Исходный длинный URL |
| `user_id` | bigint | Foreign Key | ID создателя (nullable для анонимных ссылок) |
| `created_at` | timestamp | - | Время создания записи |
| `updated_at` | timestamp | - | Время последнего изменения |

#### Таблица `click_stats` (Аналитика)
Append-only (только добавление) таблица для сбора статистики переходов. 

| Поле | Тип данных | Индекс | Описание |
| :--- | :--- | :--- | :--- |
| `id` | bigint | Primary Key | Уникальный идентификатор |
| `short_code` | varchar(15) | **Index** | Код ссылки для агрегации и группировки |
| `clicked_at` | timestamp | - | Точное время перехода (по умолчанию текущее) |
| `ip` | varchar(45) | - | IP адрес клиента (длина 45 для поддержки IPv6) |
| `user_agent` | text | - | Браузер и ОС пользователя |
| `referer` | text | - | Источник перехода (с какого сайта пришли) |
| `country` | varchar(2) | - | ISO 3166-1 alpha-2 код страны |
