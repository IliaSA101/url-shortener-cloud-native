# Документация проекта

В этой директории хранятся инженерные решения, схемы и результаты тестов.

## Структура
* **`/adr` (Architecture Decision Records)** — задокументированные причины выбора тех или иных технологий (почему FastAPI? почему RabbitMQ, а не Kafka? почему K3s?).
* **`/load-testing`** — сценарии k6/Locust и результаты нагрузочного тестирования (отчеты об RPS, задержках и потреблении ресурсов).
* **`architecture.md`** — детальное описание взаимодействия сервисов и диаграммы потоков данных.
